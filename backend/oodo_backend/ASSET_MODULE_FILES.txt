ENTITY STRUCTURE
================================================================================

Asset Entity
  - id: UUID (primary key, auto-generated)
  - assetTag: String (unique, auto-generated, format: AF-0001)
  - category: AssetCategory (ManyToOne, required)
  - serialNumber: String (unique if present, nullable)
  - acquisitionDate: LocalDate (nullable)
  - acquisitionCost: BigDecimal (nullable, for reporting only)
  - condition: String (free text, nullable)
  - location: String (required)
  - department: Department (ManyToOne, nullable)
  - status: AssetStatus (enum, default AVAILABLE)
  - isBookable: boolean (default false)
  - photoUrl: String (URL/path, nullable)
  - createdAt: Instant (auto-set via @CreationTimestamp)

AssetStatus Enum (7 states)
  - AVAILABLE
  - ALLOCATED
  - RESERVED
  - UNDER_MAINTENANCE
  - LOST
  - RETIRED
  - DISPOSED (terminal)

API ENDPOINTS
================================================================================

BASE URL: /assets

1. POST /assets
   Authorization: ASSET_MANAGER, ADMIN
   Request: AssetRegistrationRequest
   Response: HTTP 201 + AssetResponse

2. GET /assets
   Authorization: Authenticated
   Query Params: categoryId, status, location, departmentId, page, size (all optional)
   Response: HTTP 200 + Page<AssetResponse>

3. GET /assets/{id}
   Authorization: Authenticated
   Response: HTTP 200 + AssetResponse

4. PATCH /assets/{id}/status
   Authorization: ASSET_MANAGER, ADMIN
   Request: AssetStatusUpdateRequest
   Response: HTTP 200 + AssetResponse

================================================================================
STATUS TRANSITION RULES
================================================================================

State Machine (Enforced at Service Layer)

AVAILABLE → ALLOCATED, RESERVED, UNDER_MAINTENANCE, LOST, RETIRED
ALLOCATED → AVAILABLE, LOST
RESERVED → AVAILABLE, ALLOCATED
UNDER_MAINTENANCE → AVAILABLE, RETIRED
LOST → AVAILABLE, RETIRED
RETIRED → DISPOSED
DISPOSED → (terminal, no transitions)

Invalid transition example:
  RETIRED → ALLOCATED ❌ Throws InvalidStatusTransitionException
